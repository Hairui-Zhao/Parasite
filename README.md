# Parasite:面向混合作业负载的低延迟探索性作业调度框架
**对应文章第6章**

在深度学习研发集群中，探索性作业（如超参数调优、代码调试）占作业负载的绝大部分，其特点是执行时间短、数量庞大，但现有调度器通常将其与长期运行的确定性作业（如模型正式训练）同等对待，导致探索性作业面临漫长的排队延迟，严重阻碍了快速迭代的效率。

针对这一核心矛盾，我们提出了 Parasite —— 一个创新的资源管理框架。其核心思想是允许探索性作业像“寄生虫”一样，以低干扰的方式共享确定性作业已占用的GPU资源，从而大幅削减探索性作业的排队与执行时间，同时严格保障确定性作业的性能目标（如吞吐率）。

核心组件：

离线剖析器（Offline Profiler）：负责系统与作业的特征提取。它分析集群网络拓扑与设备性能，并剖析作业的动态内存波动模式、内核计算特性等关键执行特征，为干扰预测与调度决策提供数据基础。

作业管理器（Job Manager）：作为调度决策中枢。它首先利用剖析数据及内存碰撞概率模型，快速评估探索性作业与各GPU上现有确定性作业共置时可能产生的干扰。随后，结合融合估计器的性能预测，运行高效的探索性作业放置算法（EJPP），在满足多重约束（干扰、通信、负载）下，为探索性作业寻找最优的GPU放置方案。

融合估计器（Co-location Estimator）：基于剖析数据，对候选的作业共置组合进行性能建模，精准预测资源共享后的整体吞吐量变化，为作业管理器的决策提供量化依据。

内核管理器（Kernel Manager）：负责共置作业在单个GPU内的细粒度执行优化。它通过内核拦截与重排序技术，依据作业优先级及内核计算-内存访问的互补性原则，动态编排内核的执行顺序（作业融合机制， JFM），以主动缓解资源竞争，最小化性能干扰。
